# 711 — Rich stdlib reference docs with crates.io / docs.rs / JSR readability

## Summary

Generated stdlib docs should be useful as a primary API browsing surface, not
only as a manifest dump. The target is a rich reference experience inspired by
docs.rs / crates.io and JSR: clear item categories, module navigation, readable
summaries, examples, trait implementation sections, source links where
possible, and stable anchors.

This issue tracks documentation quality for the stdlib API after the type and
trait surface becomes more complete.

## Current state

- The `arukellt doc` command can generate an HTML stdlib reference.
- The reference now groups modules separately from macros, structs, enums,
  functions, and type aliases.
- The low number of structs/enums makes the stdlib look function-heavy and
  helper-heavy.
- Public docs do not yet consistently explain trait relationships,
  implementation status, examples, migration paths, or low-level helper status.

## Required work

- [ ] Define the stdlib documentation information architecture:
      Modules, Macros, Structs, Enums, Traits, Functions, Type Aliases,
      Constants, and Implementations where supported.
- [ ] Add trait pages and trait implementation sections once trait data is
      available in the manifest or doc extractor.
- [ ] Add collection type pages for `Vec<T>`, `Deque<T>` or equivalent,
      `HashMap<K, V>`, and iterator adapter structs.
- [ ] Show deprecation / internal / low-level badges for helpers that remain
      visible.
- [ ] Add examples for high-level APIs and avoid examples that teach
      `*_i32` helpers as the default style.
- [ ] Keep module navigation hierarchical and folded by default for deep trees.
- [ ] Provide local build instructions for `.build/docs/std/index.html`.
- [ ] Make GitHub Pages layout support `docs/`, `playground/`, and
      `docs/std/` without Python as a runtime dependency.

## Acceptance

- [ ] Stdlib docs are generated by `arukellt doc` or a repo command that
      delegates to it.
- [ ] Structs/enums/traits are first-class browsing categories.
- [ ] Collection ADTs appear as item pages with methods and examples.
- [ ] Low-level helpers are visually marked or hidden from the main happy path.
- [ ] Docs have stable anchors suitable for linking from issues and diagnostics.
