# 808 — T3/Wasm validation failures

## Summary

`verify quick` reports 1 aggregate failure: "T3 fixture WASM validation
gate (#686)". This expands to 213 individual fixture validation failures
where the selfhost compiler produces Wasm that fails `wasm-tools validate`.

## Exact failure scope

213 fixtures fail T3 WASM validation. The selfhost compiler emits invalid
Wasm for these fixtures. The `check-t3-wasm-validate.py` script aggregates
these into a single failure for `verify quick`.

## Machine-readable baseline

The failing fixture IDs are captured in the verify full receipt
(`docs/data/verify-full-receipt.json`) under `checks[]` where
`check_id == "t3_wasm_validate"` and `items[]` where `result == "fail"`.
The receipt is generated by `scripts/gen/write-verify-receipt.py` from
`python3 scripts/manager.py verify full` output.

The aggregate count and `pass`/`validate-fail`/`compile-fail`/`skip` totals
are in `summary.t3_*`.

## Owner

compiler team (wasm backend)

## Removal condition

Each fixture passes when the selfhost compiler emits valid Wasm that
`wasm-tools validate` accepts. The fixture is removed from the failing set
when T3 validation passes.

## Validation command

```bash
python3 scripts/check/check-t3-wasm-validate.py
```

## Current count

213 validation failures (1 aggregate in verify quick)

## New-failure ratchet

No new T3 validation failures may be added. The count must only decrease.
Any increase is a regression and blocks merge.
