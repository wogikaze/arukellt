プログラミング言語は人間が設計するため、設計判断の違いと修正の歴史を持つ。
完璧なプログラミング言語は存在しないという前提で比較すると設計意図が見えやすい。
Cでは代入が式なのでif (x = 0)は代入として評価され、意図しない条件になる。
Javaではif (x = 0)は条件式の型不一致でコンパイルエラーになる。
Pythonではx = 0を条件式に書けないため構文エラーになる。
HMMMMLでは条件式内のx = 0をx == 0として扱う設計がある。
Cの関数引数は値渡しであり、参照的な更新はポインタ値を渡して実現する。
C++は参照渡し構文を導入し、呼び出し側のオブジェクト更新を直接書ける。
Javaはオブジェクト参照を値渡しし、プリミティブ型は通常の値渡しにする。
Pythonはすべてオブジェクト参照の値渡しで、イミュータブル型により副作用を抑える。
Cではintのビット幅が環境依存で、移植時に追加確認が必要になる。
Javaはintなどのサイズと範囲を言語仕様で固定する。
C配列は長さ情報を持たず、範囲外アクセスの危険が高い。
Java配列は長さを保持し、範囲外アクセスで例外を送出する。
Java配列はオブジェクトとして扱われ、値として渡せる。
Cは関数ポインタを作成でき、高階的な呼び出しを実装できる。
Javaは関数そのものを持たない時代があり、メソッド操作はリフレクション中心だった。
C++は関数オブジェクトで呼び出し演算子をオーバーロードできる。
LispとHaskellとPythonは関数をファーストクラス値として扱う。
Pythonは引数がなくても関数呼び出しに括弧を要求する。
Lispは呼び出し構文全体を括弧で囲む。
PerlとRubyは文脈によって呼び出し括弧を省略できる。
Haskellは通常の関数適用で括弧をほとんど使わない。
OCamlは引数0個をunitで表現し、必要時に()を渡す。
PostScriptとForthは逆ポーランド記法で優先順位括弧への依存を減らす。
Smalltalkは式を左結合で評価する設計を採る。
括弧省略を許す言語では関数値そのものを参照する文法を別途整理する必要がある。
C++は演算子オーバーロードを許し、表現力と可読性のトレードオフがある。
Javaはユーザー定義演算子を制限し、x + yよりx.add(y)が増えやすい。
PythonとRubyは演算子オーバーロードを許可し、慣習で乱用を抑える。
C++の多重継承はダイヤモンド問題を生みうる。
Javaは実装継承を単一に制限し、インターフェース多重実装で補う。
RubyはMix-inで実装再利用を行う。
Pythonは多重継承を許可し、C3 linearizationで解決順序を定義する。
Perl系でもC3 MROを採用する実装がある。
LispとJavaとPythonとRubyはGCを標準機能として重視する。
CとC++はGCを前提にせず、手動管理やRAIIで性能制御を重視する。
一部言語はGCとmalloc/freeの併用を可能にして選択肢を残す。
リージョン推論だけでは速度要件を満たせずGC併用に戻る事例がある。
CとC++は未初期化を許す設計で、性能と安全性の責任を開発者に置く。
初期化強制の言語は未定義値起因のバグを減らす。
Pythonは代入でのみ変数が成立するため未初期化参照を抑制しやすい。
JavaScriptはundefinedを明示値として扱う設計で未定義状態を表現する。
Cはグローバル変数を直接定義できる。
Javaはトップレベル変数を持たず、静的メンバで近い効果を作る。
Pythonのグローバルはモジュール単位で、関数内書き換えにはglobal宣言が必要になる。
C/C++の負数整数除算の丸め方向は歴史的に実装差があった。
JavaとC99は整数除算を0方向へ丸める。
Haskellはdivとquotを分けて丸め方向を明示する。
Ruby 1.8はx / 0.0をNaNまたはInfinityで表現する。
Pythonはx / 0.0で例外を送出する。
Python 2とJavaとScalaでは0100を8進数として解釈する。
Python 3とHaskellは8進数を0o100形式で表す。
Python 2.6は100lのような小文字l接尾辞を許容した。
Dは古い版以降で小文字l接尾辞を非推奨または禁止方向にした。
PythonとRubyは整数を自動多倍長化し、オーバーフローを避ける。
CとJavaは固定幅整数を維持し、暗黙の多倍長化をしない。
JavaScriptは整数専用型を基本仕様に持たない時代が長かった。
HaskellはIntegerを標準で使えば多倍長を自然に扱える。
Pythonは変数再代入を通常機能として許す。
Rubyは定数風命名を用意するが再代入は警告止まりの実装がある。
C++とJavaはconstやfinalで再代入禁止を局所的に指定する。
Haskellは通常の束縛を再代入不可にする。
ErlangとOCamlは不変束縛中心だが可変領域を別途提供する。
Pascalは算術と比較と論理の優先順位を明確に分離する。
0 < x and x < 9のような式は言語ごとの優先順位差で結果が変わる。
Cではx == y & zが(x == y) & zとして解釈されるため意図確認が必要になる。
Pythonは0 < x <= 9の連鎖比較を一つの式として許す。
Haskellではf -1がf - 1と読まれるため負数引数にはf (-1)が必要になる。
SMLは負数リテラルに~1記法を使う。
J言語は負数を_1記法で表す。
Cはforとwhileとgotoなど複数のループ手段を持つ。
Javaはgotoを予約語にしつつ実用構文から外している。
Lispは末尾再帰最適化で再帰をループとして使える。
Haskellはmapやfoldlなどの高階関数で反復を記述する。
ML系では多相参照に関する型安全性問題が歴史的に議論された。
Cはインデントと実際の構文ブロックがずれるコードを書けてしまう。
Cでは波括弧省略時に追加行が無条件実行になる事故が起こる。
Rubyはendでブロック終端を明示する。
Pythonはインデントを構文に組み込み、見た目と構造を一致させる。
QuickBASICはIf ... End Ifで終端の対応関係を明示する。
HMMMMLはXML風タグでブロック終端を明示する。
obj.method(arg)のレシーバ受け取りは言語ごとに暗黙と明示が分かれる。
PythonとC拡張の多くはレシーバを明示引数として扱う。
Javaはthisを暗黙に提供し必要時に明示参照できる。
CLOSとHaskellではレシーバ特権を置かず通常引数として扱える。
RubyとC++の文字列は破壊的更新を許す実装がある。
PythonとJavaの文字列はイミュータブルである。
Cの文字列はchar*として低水準メモリ管理に依存する。
PerlとPHPは文字列と数値の自動変換を行う。
PerlとPHPは連結演算子と加算演算子を分けて型変換を運用する。
PythonとRubyは不適切な文字列数値混在演算を型エラーにする。
Javaは文字列連結時にtoString相当の変換を許容する。
Python 2とRubyとOCamlでは1/2が整数除算で0になる場合がある。
Python 3では1/2が浮動小数点0.5になる。
Schemeでは1/2を有理数として保持できる。
Haskellは型クラスと推論により1/2の具体型を決める。
Groovyは数値リテラルをBigDecimal寄りに扱う設計がある。
Simulaはクラスとオブジェクトの枠組みを普及させた。
Smalltalkはメッセージ送信中心の純オブジェクト観を徹底した。
C++はCの型システムを拡張してクラスと継承を導入した。
Javaはオブジェクト指向を前面に出しつつプリミティブ型を併存させた。
Java 5はオートボクシングでプリミティブとオブジェクトの橋渡しをした。
PerlとJavaScriptとPythonは辞書構造と関数値の組み合わせで柔軟なオブジェクトモデルを作る。
PerlとJavaScriptとPythonはprivateを弱く扱う文化を持つ実装が多い。
Rubyはアクセス制御を比較的重視する。
JavaとHaskellは真偽値型以外を条件式に入れない。
Cは0を偽、それ以外を真として扱う。
Schemeは#fのみ偽で0も真とする。
Rubyはfalseとnilのみ偽で0も真とする。
JavaScriptはfalseとnullとundefinedと0と空文字列とNaNを偽として扱う。
PythonはFalseとNoneと数値0と空コンテナを偽として扱う。
ALGOLとCとPascalは静的スコープを採用した。
初期Lispと初期Perlは動的スコープを使う文化があった。
Common LispとPerlは動的スコープと静的スコープの併用手段を持つ。
SchemeとPythonは静的スコープを重視する。
BASIC系にはスコープ規律が弱い方言がある。
配列範囲外アクセス時にRubyはnil、JavaScriptはundefinedを返す実装がある。
PythonとJavaは配列範囲外アクセスで例外を送出する。
Cの配列範囲外アクセスは未定義動作でクラッシュや脆弱性につながる。
