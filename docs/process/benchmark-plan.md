# ベンチマーク計画

## 目的

ADR-002（GC vs non-GC）の決定に必要なデータを収集する。

---

## 測定対象

| 指標 | 単位 | 測定方法 |
|------|------|---------|
| バイナリサイズ | bytes | `wc -c` |
| 実行時間 | ms | `hyperfine` または内部計測 |
| ピークメモリ使用量 | KB | wasmtime の `--max-wasm-stack` 出力 |

---

## ベンチマークケース

### Case 1: Hello World

**目的**: 最小バイナリサイズの比較

**仕様**:
- 標準出力に "Hello, World!" を出力
- WASI `fd_write` を使用

**成功基準**:
- Wasm GC: 2KB 以下（Wado 参考: 1090 bytes）
- linear memory: 4KB 以下（アロケータ含む）

**測定項目**:
- [ ] バイナリサイズ
- [ ] 起動時間

---

### Case 2: 文字列連結 100 回

**目的**: アロケーション負荷の比較

**仕様**:
```
result = ""
for i in 0..100:
    result = result + "x"
print(result.len())
```

**成功基準**:
- 実行時間: C 比 2.0x 以内
- メモリ使用量: 入力サイズの 3 倍以内

**測定項目**:
- [ ] 実行時間
- [ ] ピークメモリ使用量
- [ ] バイナリサイズ

---

### Case 3: Vec push/pop 10k

**目的**: コンテナ性能の比較

**仕様**:
```
vec = new Vec[i32]
for i in 0..10000:
    vec.push(i)
sum = 0
while vec.len() > 0:
    sum = sum + vec.pop()
print(sum)
```

**成功基準**:
- 実行時間: C 比 1.5x 以内

**測定項目**:
- [ ] 実行時間
- [ ] ピークメモリ使用量
- [ ] バイナリサイズ

---

### Case 4: 二分木 depth 20

**目的**: 再帰と参照の比較

**仕様**:
- 深さ 20 の完全二分木を構築
- 全ノードの合計値を計算
- 各ノードは (left, right, value) を持つ

**成功基準**:
- 実行時間: C 比 1.5x 以内
- メモリ使用量: ノード数 × ノードサイズ × 2 以内

**測定項目**:
- [ ] 実行時間
- [ ] ピークメモリ使用量
- [ ] バイナリサイズ

---

### Case 5: Result ベースエラー処理

**目的**: エラー経路のオーバーヘッド測定

**仕様**:
```
fn fallible(n: i32) -> Result[i32, Error]:
    if n < 0:
        return Err(Error::Negative)
    else:
        return Ok(n * 2)

sum = 0
for i in -1000..1000:
    match fallible(i):
        Ok(v) => sum = sum + v
        Err(_) => continue
print(sum)
```

**成功基準**:
- 実行時間: エラーなし版と比較して 1.2x 以内

**測定項目**:
- [ ] 実行時間（エラーあり）
- [ ] 実行時間（エラーなし比較）
- [ ] バイナリサイズ

---

### Case 6: WASI ファイル読込

**目的**: I/O 経路の比較

**仕様**:
- 1KB のテストファイルを読み込む
- 内容のバイト数を出力

**成功基準**:
- 実行時間: ファイルアクセス時間 + 1ms 以内

**測定項目**:
- [ ] 実行時間
- [ ] バイナリサイズ

---

## 実装方式

### Wasm GC 版 (`harness/proto/gc/`)

- `ref`, `struct`, `array` 型を使用
- ホスト GC に管理を委ねる
- メモリ管理ルーチンを含まない

### linear memory 版 (`harness/proto/linear/`)

- linear memory 上に自前アロケータ実装
- bump allocator（最も単純な実装）
- 解放は行わない（ベンチマーク用の簡易実装）

---

## 実行環境

- **ランタイム**: wasmtime 28.0+ (Wasm GC サポート必須)
- **ホスト**: Linux x86_64
- **反復回数**: 各ケース 10 回実行、中央値を採用

---

## 判定基準

| 条件 | 決定 |
|------|------|
| GC 版が全ケースで linear 版と同等以上 | GC を採用 |
| linear 版が 2 ケース以上で GC 版の 1.5x 以上高速 | linear を採用 |
| それ以外 | LLM フレンドリを優先して GC を採用 |

「同等」の定義: 実行時間差 10% 以内、バイナリサイズ差 20% 以内

---

## スケジュール

1. Case 1-2 を先行実装（最小限で方向性を確認）
2. 結果が明確な場合はここで決定
3. 不明確な場合は Case 3-6 を追加実装
