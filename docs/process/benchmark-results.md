```bash

mise bench:quick            # single-sample smoke run
mise bench:compare          # measure + compare against baseline

mise bench:ci               # regression gate (fail on threshold breach)
```

## Cross-language comparison

Native reference programs (C, Rust, Go) are built with each toolchain’s release-style flags, timed with `hyperfine` when available (otherwise a built-in shell timer), and compared to Ark wasm. `bash scripts/compare-benchmarks.sh` prints the table to **stdout** and embeds the same Markdown table in the block below (between HTML comments so `scripts/util/benchmark_runner.py` can regenerate the rest of this file without deleting it).

```bash

# equivalent:

```

## Schema

Current benchmark JSON uses `schema_version = arukellt-bench-v1` and stores:
- run metadata (`mode`, `target`, `environment`, tool availability)
- per-benchmark `compile` metrics (`median_ms`, `binary_bytes`, `max_rss_kb`)
- per-benchmark `runtime` metrics (`median_ms`, `p50_ms`, `p95_ms`, `p99_ms`, `stddev_ms`, `startup_ms`, `guest_ms`, `max_rss_kb`, correctness)
- per-benchmark `memory` metrics (`compiler_rss_peak_kb`, `runtime_rss_peak_kb`, `gc_pause_*` — GC fields are `"unavailable"` because wasmtime CLI does not expose GC pause telemetry)
- workload tags for taxonomy and future grouped reporting
