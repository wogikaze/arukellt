## 目的

`docs/rfcs/` は、ADR に載せるには大きすぎる**設計提案・仕様草案・移行設計**を置く場所である。

| 置き場 | 役割 |
|--------|------|
| `docs/adr/` | 採択した判断（なぜ選んだか）。短く固定する。 |
| `docs/rfcs/` | 詳細仕様・型設計・不変条件・代替案の長文。可変でよい。 |
| `docs/plans/` | フェーズ・PR 分割・チェックリストなど実装計画。 |
| `issues/` | 作業追跡・完了条件・担当境界。 |
| `docs/current-state.md` | 現行挙動の正本（件数・対応状況など）。 |
| `docs/design/` | 上記に分類しきれない設計メモ・比較検討。 |

## ファイル命名

`NNN-kebab-slug.md`（例: `001-typed-mir-spine.md`）。番号は RFC 専用で ADR 番号と独立。

## 推奨構成

1. 要約
2. 動機 / 問題
3. 提案
4. 詳細設計
5. 代替案と却下理由
6. 移行・互換
7. 未決事項
8. 関連 ADR / issue

採択された中核判断だけを ADR に残し、詳細は本ディレクトリの RFC を参照する。
