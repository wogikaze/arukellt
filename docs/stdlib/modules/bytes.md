## Overview

> **Overview vs Reference:** This section is curated prose — it explains when and how to use this module family. The sections below are exhaustive generated reference tables sourced directly from `std/manifest.toml` and source doc comments.

The `std::bytes` module provides binary data helpers built on `Vec<i32>`. Each element is treated as a byte in the `0..=255` range. The module covers buffer creation and manipulation, string-to-bytes conversion, hex encoding/decoding, LEB128 variable-length encoding, and little-/big-endian integer conversions.

**Recommended API highlights:**

| API | Purpose |
|-----|---------|
| `bytes_from_string(s)` / `string_from_bytes(buf)` | Convert between UTF-8 strings and byte buffers. |
| `hex_encode(buf)` / `hex_decode(s)` | Hex-encode a buffer or decode a hex string. |
| `leb128_encode_u32(n)` / `leb128_encode_i32(n)` | Variable-length LEB128 encoding. |
| `u32_to_le_bytes(n)` / `u32_from_le_bytes(buf)` | Little-endian u32 ↔ byte conversion. |
| `bytes_slice(buf, start, end)` | Extract a sub-range of bytes as a new buffer. |
| `bytes_concat(a, b)` | Concatenate two byte buffers into a new one. |

**Target constraints:** All targets. No host capability required.

**Typical usage:**

```ark
import std::bytes

let buf = bytes_from_string("hello")
let hex = hex_encode(buf)        // "68656c6c6f"
let back = hex_decode(hex)       // [104, 101, 108, 108, 111]
let s = string_from_bytes(back)  // "hello"
```
